Check databases for duplicates

MarDB database

$ cat mardb_nucleotides_V6.fna | seqkit rmdup -n -o mardbnoDup_nucleotides.fna -d duplicates_mardb.fna -D duplicates_mardb_info.txt 1>log_rmdup.txt 2>&1 &

prepare a single fasta

Join the two databases in a single fasta:

$ cat marref_nucleotides_V6.fna mardbnoDup_nucleotides.fna > mar_nucleotides.fna &

Rename sequences:

Custom python script, first version: rename_mar_seqs.py

from Bio.SeqIO.FastaIO import SimpleFastaParser

newseqs = {}
equivalent = {}
# parse fasta file with the low-level SimpleFastaParser, reads it as a tuple
with open("mar_nucleotides.fna") as sequences:
	for k, seq in enumerate(SimpleFastaParser(sequences)):
		newseqs[k]=seq[1]
		equivalent[k]=seq[0]


ofile = open("mar_renamed_nucleotides.fna", "w")
for i in newseqs.keys():
	ofile.write(">{}\n{}\n".format(i, newseqs[i]))
ofile.close()


ofile = open("mar_names_equivalence.txt", "w")
for i in equivalent.keys():
	ofile.write("{}\t{}\n".format(i, equivalent[i]))
ofile.close()

Copy to your folder where the fasta file is, check permisions with chmod and run it:

$ chmod u+x rename_mar_seqs.py
$ python rename_mar_seqs.py 1>log_rename.txt 2>&1 &

Using HMMs to fish out of our protein (NosZ) from reference database.

Download hmm files:

$ wget 'https://ftp.ncbi.nlm.nih.gov/hmm/current/hmm_PGAP.HMM/TIGR04246.1.HMM'
$ wget 'https://ftp.ncbi.nlm.nih.gov/hmm/current/hmm_PGAP.HMM/TIGR04244.1.HMM'

To use both HMM, to put them toguether in a single file.

$ cat TIGR04244.1.HMM TIGR04246.1.HMM > TIGRnosZ.HMM

Now, we can use HMMER to search for nosZ in the database. We are using the gathering score from the models. With the sample database:

hmmsearch --noali --domtblout marsample_nosZ_tigrfam.hmm --cut_ga --cpu 15 TIGRnosZ.HMM marsample_proteintrans_clean.faa > /dev/null 2>hmmerrlog.txt &

Full database:

hmmsearch --noali --domtblout /mnt/nfelbrus2/mar_db/mar_nosZ_tigrfam.hmm --cut_ga --cpu 15 /media/disk5/nfernandez/nosZ/TIGRnosZ/TIGRnosZ.HMM /mnt/nfelbrus2/mar_db/mar_proteintrans_clean.faa > /dev/null 2>/mnt/nfelbrus2/mar_db/hmmerrlog.txt &

Filter nosZ database to retrieve hits’ sequences:

Once it is done, pick up sequence IDs from hmm output table with the python script. This script produces a text file with hit´s IDs

parse_marsampleHMM.py

from Bio import SearchIO
import csv

# Read the output table of database search with TIGRFAMs HMMs for nosZ
hmm_qresult = SearchIO.parse('marsample_nosZ_tigrfam.hmm',  'hmmsearch3-domtab')

# Filter table to make a list of hits IDs
hit_ids = []
for qresult in hmm_qresult:
    for i in range(len(qresult)):
        hit_ids.append(qresult[i].id)

# Write list to a text file
file = open('hits_nosZ_marsample.txt', 'w')
for index in range(len(hit_ids)):
    file.write(str(hit_ids[index]) + "\n")
file.close()

Let´s run it and have a look:

$ python parse_marsampleHMM.py
$ less hits_nosZ_marsample.txt

Now, filter database using seqkit tools to keep sequences included in hits_nosZ_marsample.txt

$ seqkit grep -f hits_nosZ_marsample.txt marsample_proteintrans_clean.faa -o hits_nosZ_marsample.faa

This step produces a fasta file, hits_nosZ_marsample.faa, which contains the putative nosZ sequences only.

Commands to work with the full database are similar, just change names/paths also in the python script:

$ python parse_marNosZhmm.py
$ wc -l hits_mar_nosZ.txt

Now, align nosZ squences:

$ mkdir alignments

Phylogenetic placement of environmental sequences

Alignment of environmental sequences with PaPaRa

Run PaPaRa with the -r option to ensure no additional gaps are included in the original MSA. Other options are -a for aminoacid sequences, -t reference tree, -s reference alignment -q sequences to align (query sequences), -j threads, -n sufix for output:

$ cd alignments/
$ papara -t hits_nosZ_aligned.phy.treefile -a -r -s hits_nosZ_aligned.phy -q ../query.faa -j 4 -n papout 1>log_papara.txt 2>&1 &

PaPaRa outputs a MSA including both the reference and query sequences, but EPA-ng needs them separated, it is easy to separate:

$ epa-ng --split hits_nosZ_aligned.phy papara_alignment.papout
INFO Splitting files based on reference: hits_nosZ_aligned.phy

it produces two files: reference.fasta and query.fasta which have the aligned sets.

Phylogenetic placement of environmental sequences with EPA-ng

Pace the query sequences in the reference tree with EPA-ng:

Make output directory first and use the MSAs in fasta format produced by epa-ng --split.

We have to indicate the substitution model used in the three.

$ mkdir epa-ng-test
$ epa-ng --tree hits_nosZ_aligned.phy.treefile --ref-msa reference.fasta --query query.fasta --out-dir epa-ng-test --model LG+F+R10 > log_epang.txt 2>&1 &

Now, we can start analyzing the results with gappa

$ gappa examine graft --jplace-path epa_result.jplace --name-prefix FUNGEN --out-dir gappaout
